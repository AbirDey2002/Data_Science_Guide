Part 1 — Model Training, Theory & Concepts

Why we train, loss functions, optimisers, regularisation, evaluation, hyperparameter tuning, and the full sklearn workflow.

Why We Train

Training is the process of finding the best mathematical function that maps inputs X to outputs y — automatically from data, rather than hand-coding rules.

Core Idea

  • Model training = finding parameters θ such that f(X) ≈ y
  • Parameters are learned by minimising a loss function
  • The real goal is generalisation — performing well on unseen data, not memorising training data
  • Model choice = choosing what shape of function to search over (line, tree, etc.)

Three Pillars

Generalisation

Train so model works on data it has never seen. Not memorisation.

Bias-Variance Tradeoff

Central tension between underfitting and overfitting.

Loss Minimisation

Training mechanically = minimising a number (loss) that measures wrongness.

Training Loop (Mental Model)

X (features) → [ Model θ ] → ŷ (prediction)
                                    ↕
                               y (truth)
                          → Loss = how wrong?
                          → Adjust θ, repeat

Bias-Variance Tradeoff

Every model's error decomposes into three parts. Understanding this decomposition drives almost every ML decision.

$$\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

Definitions

TermMeaningSymptomCalled
BiasError from wrong assumptions — model too simpleBad on train AND testUnderfitting
VarianceError from sensitivity to training data — model too complexGood on train, bad on testOverfitting
Irreducible NoiseRandom noise baked into data — cannot be removedIrreducible floor—

The Trade-off

Simple model  → High Bias,   Low Variance   (underfits)
Complex model → Low Bias,    High Variance  (overfits)
Sweet spot    → bottom of U-shaped total error curve

Practical Diagnostic

High Bias

Model bad on TRAINING data → try more complex model, add features

High Variance

Model bad on TEST data only → regularise, simplify, get more data
More data reduces variance but does NOT fix bias. If the model is structurally wrong, more data won't help.

Loss Functions

A loss function converts model wrongness into a single differentiable number. Training minimises this number. Choosing the wrong loss means optimising for the wrong thing.

Regression Losses

LossFormulaKey PropertyUse When
MSE$(1/n)\sum(y-\hat{y})^2$Squares errors — punishes large errors heavilyLarge errors are truly bad; no outliers
MAE$(1/n)\sum|y-\hat{y}|$Absolute errors — robust to outliersOutliers present
HuberMSE for small errors, MAE for largeBest of both — smooth + robustWant robustness AND smooth gradients

Classification Losses

LossFormulaKey PropertyUse When
Log Loss (BCE)$-(1/n)\sum[y\log\hat{y}+(1-y)\log(1-\hat{y})]$Massively penalises confident wrong predictionsBinary classification — almost always
Loss function vs Evaluation Metric — different things. You train with Log Loss but report Accuracy or F1 to stakeholders. Never confuse these.

Loss vs Metric

Loss FunctionEvaluation Metric
PurposeDrives training (must be differentiable)Measures real-world performance
ExamplesMSE, Log Loss, HuberRMSE, Accuracy, F1, AUC
Used whenDuring training loopAfter training, to judge model

Gradient Descent

The algorithm that minimises the loss by iteratively nudging weights in the direction that reduces error. The "engine" that makes training work.

The Update Rule

$$\theta = \theta - \alpha \cdot \nabla L(\theta)$$

θ = weights | α = learning rate | ∇L = gradient of loss

Learning Rate

α too large  → overshoots minimum → loss bounces / explodes
α too small  → training takes forever
α just right → steady descent to minimum
Typical values: 0.01, 0.001, 0.0001

Three Flavours

TypeData per UpdatePropertyUsed?
Batch GDFull datasetAccurate gradient, very slowRarely
Stochastic GD1 sampleFast, noisy gradientSometimes
Mini-Batch GD32–256 samplesBalance of speed + accuracyStandard

One Training Loop

1. Initialise weights randomly (θ)
2. For each epoch:
      For each mini-batch:
          a. Forward pass  → compute ŷ
          b. Compute loss  → L(y, ŷ)
          c. Backward pass → compute gradients ∇L
          d. Update weights → θ = θ - α∇L
3. Repeat until loss stops improving
Local minima exist but are less of a problem in practice than they sound — mini-batch noise and modern optimisers help escape them.

Sigmoid Derivative — Why ŷ(1-ŷ)

$$\frac{d\sigma}{dz} = \sigma(z)(1 - \sigma(z)) = \hat{y}(1-\hat{y})$$

Derived via chain rule on $(1+e^{-z})^{-1}$. Largest at ŷ=0.5, vanishes at extremes — seed of the vanishing gradient problem.

Log Loss Gradient — The Beautiful Cancellation

$$\frac{\partial L}{\partial \theta_j} = \frac{1}{n}\sum(\hat{y}_i - y_i) \cdot x_{ij}$$

Chain rule: ∂L/∂ŷ × ∂ŷ/∂z × ∂z/∂θ — the sigmoid and log loss cancel to just (ŷ - y)×x

Regularisation — Fixing Overfitting

Any technique that penalises model complexity to prevent overfitting. Adds a penalty term to the loss so the model balances fitting data vs staying simple.

$$\text{New Loss} = \text{Original Loss} + \lambda \cdot \text{Complexity Penalty}$$

λ (lambda) controls how hard complexity is penalised

L1 vs L2

L2 — RidgeL1 — LassoElastic Net
Penalty$\lambda\sum\theta^2$$\lambda\sum|\theta|$Both combined
Gradient$2\lambda\theta$ — shrinks proportionally$\lambda \cdot \text{sign}(\theta)$ — fixed step—
EffectWeights small but never zeroSome weights exactly zeroBoth effects
Feature selectionNoYes — automaticYes
Use whenMany correlated featuresMany irrelevant featuresUnsure which

Why L1 Can Zero Out Weights but L2 Cannot

L2 Update

θ = θ × (1 - 2αλ)

Multiplies θ by fraction of itself each step. Push gets weaker as θ → 0. Asymptotically approaches zero, never reaches it. Geometrically: circle constraint — no corners.

L1 Update

θ = θ - α·λ·sign(θ)

Subtracts a FIXED constant regardless of size of θ. When θ is tiny, still subtracts the same amount — blows through zero. Geometrically: diamond constraint — corners sit on axes.

λ (Regularisation Strength)

λ = 0        → no regularisation → free to overfit
λ very large → model too simple → underfits
λ just right → balanced ✅
Note: sklearn uses C = 1/λ — so high C = less regularisation

Other Regularisation Techniques

  • Early Stopping — stop training when validation loss starts rising
  • Dropout — randomly zero out neurons during training (neural nets)
  • Cross-Validation — more honest generalisation estimate
Never regularise the bias term (θ₀ / intercept). Only regularise feature weights. In Ridge's normal equation: set I[0,0] = 0 before adding λI.

Train / Validation / Test Splits + Cross-Validation

Proper evaluation requires held-out data the model never saw. Two train/test is not enough — you need three sets or cross-validation to avoid leaking test information into decisions.

Three-Way Split

Full Dataset
├── Training Set   (60-70%)  → model learns from this
├── Validation Set (15-20%)  → tune hyperparameters on this
└── Test Set       (15-20%)  → FINAL evaluation — touched ONCE only
The Golden Rule: Touch the test set exactly once, at the very end. Never tune based on it. Tuning on test = data leakage.

K-Fold Cross-Validation

Rotate which fold is validation set k times. Average the k scores for a stable, trustworthy estimate. Operates entirely within training data — test set stays locked.

from sklearn.model_selection import cross_val_score, KFold

# Classification (auto-stratified)
cv_scores = cross_val_score(model, X_train, y_train, cv=5, scoring='roc_auc')

# Regression
cv_rmse = cross_val_score(model, X_train, y_train, cv=5,
                          scoring='neg_root_mean_squared_error')
rmse = -cv_rmse.mean()   # flip sign back — sklearn negates for maximisation

print(f"Mean: {cv_scores.mean():.3f}  Std: {cv_scores.std():.3f}")

Reading CV Results

Low Std — Good

Stable across folds. Model is reliable and generalises well.

High Std — Warning

Unstable — sensitive to which data it sees. High variance signal.

Why neg_ prefix in sklearn scoring?

sklearn always maximises. RMSE/MAE are errors (minimise). Negating converts minimisation to maximisation. Always flip sign back when reading results.

Stratified K-Fold

For classification — ensures each fold has the same class ratio. sklearn uses this automatically for classifiers. For regression, plain KFold is used.

Data Leakage — Critical Warning

When information from the test set (or future) leaks into training or preprocessing, producing falsely optimistic results that fail in production.

Most Common Leakage Mistake

Wrong — Leakage

scaler.fit_transform(X)  # full dataset
# test set statistics leak into scaler

Correct — No Leakage

scaler.fit_transform(X_train)  # train only
scaler.transform(X_test)       # apply same

Pipeline — The Safe Pattern

from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model',  LogisticRegression())
])

pipe.fit(X_train, y_train)
pipe.score(X_test, y_test)  # ALWAYS score through pipeline, not model directly
Calling model.score(X_test) directly bypasses the pipeline's scaler — passing raw unscaled data to a model trained on scaled data. Always use pipe.score(), never model.score().

Other Leakage Sources

  • Fitting encoders, imputers, or any preprocessor on full data before splitting
  • Using future data as features (time-series)
  • Tuning hyperparameters by repeatedly checking test set

Evaluation Metrics

Accuracy alone is dangerous with imbalanced classes. Always pick the metric aligned with your actual business goal.

The Confusion Matrix

                    Predicted
                  Positive  Negative
Actual  Positive │   TP   │   FN   │
        Negative │   FP   │   TN   │

TP = True Positive  | TN = True Negative
FP = False Positive (Type I)  | FN = False Negative (Type II)

Classification Metrics

MetricFormulaUse When
Accuracy(TP+TN)/(TP+TN+FP+FN)Balanced classes, equal error cost
PrecisionTP/(TP+FP)False positives costly (spam filter)
RecallTP/(TP+FN)False negatives costly (cancer screening)
F1 Score2×(P×R)/(P+R)Imbalanced classes, balance P and R
AUC-ROCArea under ROC curveCompare models regardless of threshold
Precision and Recall always trade off. Being more aggressive catches more positives (Recall↑) but causes more false alarms (Precision↓). Your business problem decides which matters more.

Regression Metrics

MetricFormulaUse When
RMSE$\sqrt{\text{mean}(y-\hat{y})^2}$General purpose — same units as target
MAE$\text{mean}(|y-\hat{y}|)$Outliers present — more robust
R²$1 - SS_{res}/SS_{tot}$Explain variance — stakeholder communication

Reading RMSE

RMSE is in same units as target (e.g. dollars for salary)
Always contextualise: RMSE / mean(y) × 100 = % error
Compare to baseline: np.sqrt(np.mean((y_train - y_train.mean())**2))
If model RMSE > baseline RMSE → model is worse than predicting mean ❌

R² Interpretation

R² = 1.0  → perfect prediction
R² = 0.0  → no better than predicting the mean
R² < 0.0  → worse than predicting the mean (model is broken)

Full Tuning Workflow

1. Train with defaults → establish baseline
2. RandomizedSearch with wide ranges → explore broadly
3. GridSearch in promising region → zoom in
4. Retrain best model on ALL training data
5. Evaluate ONCE on test set → final number
Pipeline + GridSearch is the production pattern. Use 'step_name__param_name' syntax for pipeline parameters. This prevents leakage during CV.

Linear Regression vs Ridge vs Lasso

All three fit the same straight line ŷ = θ₀ + θ₁x₁ + ... The ONLY difference is what they minimise during training.

Linear RegressionRidge (L2)Lasso (L1)
Loss$\sum(y-\hat{y})^2$$\sum(y-\hat{y})^2 + \lambda\sum\theta^2$$\sum(y-\hat{y})^2 + \lambda\sum|\theta|$
PenaltyNoneSquared weightsAbsolute weights
WeightsUnconstrainedShrunk, never zeroSome exactly zero
Feature selectionNoNoYes — automatic
HyperparameterNonealpha (λ)alpha (λ)
Use whenFew clean features, baselineMany correlated featuresMany irrelevant features
All three are the same model — same line equation. Only the loss function changes. Ridge and Lasso are Linear Regression with a penalty added.

Why L1 Zeros Weights But L2 Cannot — The Math

L2 Gradient → proportional push

Penalty gradient = $2\lambda\theta$

Update: $\theta = \theta(1 - 2\alpha\lambda)$

As θ → 0, push gets weaker. Asymptotically approaches zero.

L1 Gradient → fixed step

Penalty gradient = $\lambda \cdot \text{sign}(\theta)$

Update: $\theta = \theta - \alpha\lambda\cdot\text{sign}(\theta)$

Fixed constant subtracted regardless of θ size. Crosses zero.

Geometric Intuition

L2 constraint region → CIRCLE  (smooth, no corners)
  → loss contours touch circle at a smooth point → both weights non-zero

L1 constraint region → DIAMOND (sharp corners on the axes)
  → loss contours likely hit a corner → one weight = exactly zero

Linear Regression — From Scratch

Finds θ that minimises MSE. Two methods: Normal Equation (direct, one shot) and Gradient Descent (iterative).

Normal Equation

$$\theta = (X^TX)^{-1}X^Ty$$

Direct closed-form solution. O(p³) — breaks for large p. Fails if XᵀX is not invertible.

Gradient of MSE

$$\frac{\partial \text{MSE}}{\partial \theta_0} = \frac{-2}{n}\sum(y - \hat{y}) \qquad \frac{\partial \text{MSE}}{\partial \theta_1} = \frac{-2}{n}\sum(y-\hat{y}) \cdot x$$ 
class LinearRegression:
    def __init__(self, learning_rate=0.01, n_iterations=1000):
        self.learning_rate = learning_rate
        self.n_iterations  = n_iterations
        self.theta_0 = None   # intercept
        self.theta_1 = None   # slope
        self.loss_history = []

    def fit(self, X, y):
        n = X.shape[0]
        self.theta_0 = 0.0
        self.theta_1 = 0.0

        for i in range(self.n_iterations):
            y_hat        = self.theta_0 + self.theta_1 * X   # forward pass
            errors       = y - y_hat                          # residuals
            loss         = (errors ** 2).mean()               # MSE
            self.loss_history.append(loss)

            grad_0 = (-2 / n) * errors.sum()                  # ∂MSE/∂θ₀
            grad_1 = (-2 / n) * (errors * X).sum()            # ∂MSE/∂θ₁

            self.theta_0 = self.theta_0 - self.learning_rate * grad_0
            self.theta_1 = self.theta_1 - self.learning_rate * grad_1

    def predict(self, X):
        return self.theta_0 + self.theta_1 * X

    def score(self, X, y):
        y_hat  = self.predict(X)
        ss_res = ((y - y_hat) ** 2).sum()
        ss_tot = ((y - y.mean()) ** 2).sum()
        return 1 - (ss_res / ss_tot)   # R²
Ridge extension: add 2*lambda*theta to grad_1 (not grad_0 — never regularise the bias). For Normal Equation: add λI with I[0,0]=0.

Interview Checklist

  • Prepend bias column of 1s (matrix version)
  • Know Normal Equation and when it breaks (>10k features)
  • Gradient formula: (−2/n) × errors.sum() for θ₀
  • Track loss history — shows understanding of training loop
  • score() returns R², not just fit()

Logistic Regression — From Scratch

Classification model (despite the name). Squashes linear output through sigmoid to get probability. Minimises Log Loss via gradient descent.

Key Formulas

$$z = \theta_0 + \theta_1x_1 + \theta_2x_2 \quad \hat{y} = \sigma(z) = \frac{1}{1+e^{-z}}$$ $$L = -\frac{1}{n}\sum[y\log\hat{y} + (1-y)\log(1-\hat{y})]$$ $$\frac{\partial L}{\partial \theta_j} = \frac{1}{n}\sum(\hat{y}_i - y_i) \cdot x_{ij}$$ 
class LogisticRegression:
    def __init__(self, learning_rate=0.1, n_iterations=1000):
        self.learning_rate = learning_rate
        self.n_iterations  = n_iterations
        self.theta_0 = None
        self.theta   = None   # one weight per feature
        self.loss_history = []

    def _sigmoid(self, z):
        return 1 / (1 + np.exp(-z))   # squash z → (0,1)

    def fit(self, X, y):
        n_samples, n_features = X.shape[0], X.shape[1]
        self.theta_0 = 0.0
        self.theta   = np.zeros(n_features)   # [0.0, 0.0] for 2 features

        for i in range(self.n_iterations):
            z     = self.theta_0 + np.dot(X, self.theta)   # linear combination
            y_hat = self._sigmoid(z)                         # probabilities

            term_1 = y * np.log(y_hat)                       # log loss
            term_0 = (1 - y) * np.log(1 - y_hat)
            loss   = -1 * (term_1 + term_0).mean()
            self.loss_history.append(loss)

            errors  = y_hat - y                              # (ŷ - y)
            grad_0  = (1 / n_samples) * errors.sum()         # ∂L/∂θ₀
            grad_th = (1 / n_samples) * np.dot(X.T, errors)  # ∂L/∂θⱼ

            self.theta_0 = self.theta_0 - self.learning_rate * grad_0
            self.theta   = self.theta   - self.learning_rate * grad_th

    def predict_proba(self, X):
        z = self.theta_0 + np.dot(X, self.theta)
        return self._sigmoid(z)

    def predict(self, X, threshold=0.5):
        return (self.predict_proba(X) >= threshold).astype(int)

    def score(self, X, y):
        return (self.predict(X) == y).sum() / len(y)

Key Differences vs Linear Regression

Linear RegressionLogistic Regression
Forward passθ₀ + θ₁xsigmoid(θ₀ + Xθ)
LossMSELog Loss
Gradient sign-2/n+1/n
Outputcontinuous numberprobability → class
Starting Log Loss is always 0.693 (-log(0.5)) when weights initialised to zero — maximum uncertainty. It can only go down from there.

KNN — From Scratch

Lazy learner — no training. Memorises data. At prediction time, finds K nearest neighbours and takes a majority vote (classification) or average (regression).

Euclidean Distance

$$d(a, b) = \sqrt{\sum_i (a_i - b_i)^2}$$ 
class KNN:
    def __init__(self, k=3):
        self.k       = k
        self.X_train = None
        self.y_train = None

    def fit(self, X, y):
        self.X_train = X   # just memorise — no training
        self.y_train = y

    def _euclidean_distance(self, point_a, point_b):
        difference = point_a - point_b      # element-wise subtract
        squared    = difference ** 2         # square each diff
        summed     = squared.sum()           # sum all
        return np.sqrt(summed)               # square root

    def _predict_one(self, x):
        distances = []
        for i in range(len(self.X_train)):
            dist = self._euclidean_distance(x, self.X_train[i])
            distances.append((dist, i))

        distances.sort(key=lambda pair: pair[0])   # sort nearest first
        k_nearest = distances[:self.k]              # take K closest

        k_labels   = [self.y_train[idx] for dist, idx in k_nearest]
        k_labels   = np.array(k_labels)

        n_class_1  = (k_labels == 1).sum()
        n_class_0  = (k_labels == 0).sum()
        return 1 if n_class_1 > n_class_0 else 0   # majority vote

    def predict(self, X):
        return np.array([self._predict_one(X[i]) for i in range(len(X))])

    def score(self, X, y):
        return (self.predict(X) == y).sum() / len(y)

K Trade-off

K = 1   → nearest only → very sensitive → overfits
K = n   → all vote → always predicts majority class → underfits
Small K → low bias,  high variance
Large K → high bias, low variance
Scaling is CRITICAL for KNN. Distance is meaningless if features are on different scales. A salary feature will completely dominate hours studied without normalisation.

Interview Checklist

  • fit() just memorises — no weight learning
  • Euclidean distance formula from scratch
  • Sort distances, pick K smallest
  • Majority vote (classification) or mean (regression)
  • Prediction is O(n) — slow for large datasets
  • Always scale before KNN

Decision Tree — From Scratch

Recursively splits data using the feature+threshold that maximises Information Gain. Builds a binary tree of decisions until base cases are hit.

Entropy & Information Gain

$$H = -\sum p_i \log_2(p_i)$$ $$IG = H(\text{parent}) - \frac{n_L}{n}H(\text{left}) - \frac{n_R}{n}H(\text{right})$$

H=0 → perfectly pure. H=1 → maximum disorder (50/50). Pick split with highest IG.

class Node:
    def __init__(self):
        self.feature_index = None   # which feature to split on
        self.threshold     = None   # split value
        self.left          = None   # left child (≤ threshold)
        self.right         = None   # right child (> threshold)
        self.prediction    = None   # set if leaf node

    def is_leaf(self):
        return self.prediction is not None

class DecisionTree:
    def __init__(self, max_depth=3, min_samples_split=2):
        self.max_depth         = max_depth
        self.min_samples_split = min_samples_split
        self.root              = None

    def _entropy(self, y):
        n = len(y)
        if n == 0: return 0
        p_1 = (y == 1).sum() / n
        p_0 = (y == 0).sum() / n
        if p_1 == 0 or p_0 == 0: return 0.0
        return -(p_0 * np.log2(p_0)) - (p_1 * np.log2(p_1))

    def _information_gain(self, y_parent, y_left, y_right):
        h_parent = self._entropy(y_parent)
        n = len(y_parent)
        if len(y_left) == 0 or len(y_right) == 0: return 0
        h_children = (len(y_left)/n * self._entropy(y_left) +
                      len(y_right)/n * self._entropy(y_right))
        return h_parent - h_children

    def _best_split(self, X, y):
        best_ig, best_feat, best_thresh = -1, None, None
        for feat_idx in range(X.shape[1]):
            for threshold in np.unique(X[:, feat_idx]):
                left_mask  = X[:, feat_idx] <= threshold
                right_mask = X[:, feat_idx] >  threshold
                ig = self._information_gain(y, y[left_mask], y[right_mask])
                if ig > best_ig:
                    best_ig, best_feat, best_thresh = ig, feat_idx, threshold
        return best_feat, best_thresh, best_ig

    def _build_tree(self, X, y, depth):
        # Base cases
        if self._entropy(y) == 0:               # pure node
            leaf = Node(); leaf.prediction = int(y[0]); return leaf
        if depth >= self.max_depth:              # max depth
            leaf = Node()
            leaf.prediction = 1 if (y==1).sum() > (y==0).sum() else 0
            return leaf
        if len(y) < self.min_samples_split:      # too few samples
            leaf = Node()
            leaf.prediction = 1 if (y==1).sum() > (y==0).sum() else 0
            return leaf

        best_feat, best_thresh, best_ig = self._best_split(X, y)
        if best_ig == 0:
            leaf = Node()
            leaf.prediction = 1 if (y==1).sum() > (y==0).sum() else 0
            return leaf

        node = Node()
        node.feature_index = best_feat
        node.threshold     = best_thresh
        left_mask  = X[:, best_feat] <= best_thresh
        right_mask = X[:, best_feat] >  best_thresh
        node.left  = self._build_tree(X[left_mask],  y[left_mask],  depth+1)
        node.right = self._build_tree(X[right_mask], y[right_mask], depth+1)
        return node

    def fit(self, X, y):
        self.root = self._build_tree(X, y, depth=0)

    def _predict_one(self, x, node):
        if node.is_leaf(): return node.prediction
        if x[node.feature_index] <= node.threshold:
            return self._predict_one(x, node.left)
        return self._predict_one(x, node.right)

    def predict(self, X):
        return np.array([self._predict_one(X[i], self.root) for i in range(len(X))])

    def score(self, X, y):
        return (self.predict(X) == y).sum() / len(y)

Three Base Cases (Must Know)

1. Pure Node

All samples same class → leaf, predict that class

2. Max Depth

Reached limit → leaf, predict majority class

3. Too Few Samples

Below min_samples_split → leaf, predict majority class
Gini is default in sklearn (not entropy). Same concept, different formula: Gini = 1 - Σpᵢ². Slightly faster (no log). Results nearly identical in practice.

Naive Bayes — From Scratch

Uses Bayes theorem directly. No gradient descent. fit() computes priors, means, variances per class. predict() applies Gaussian PDF per feature and picks highest posterior.

Bayes Theorem

$$P(\text{class}|\text{features}) \propto P(\text{features}|\text{class}) \times P(\text{class})$$

Posterior ∝ Likelihood × Prior. Evidence P(features) is same for all classes — ignored when comparing.

Gaussian PDF

$$P(x_j|\text{class}) = \frac{1}{\sqrt{2\pi\sigma^2}}\exp\!\left(-\frac{(x_j - \mu)^2}{2\sigma^2}\right)$$

μ and σ² are learned per feature per class during fit()

class GaussianNaiveBayes:
    def __init__(self):
        self.classes   = None
        self.priors    = {}   # P(class)
        self.means     = {}   # mean per feature per class
        self.variances = {}   # variance per feature per class

    def fit(self, X, y):
        n_samples     = X.shape[0]
        self.classes  = np.unique(y)
        for c in self.classes:
            class_mask       = (y == c)
            X_class          = X[class_mask]
            self.priors[c]   = X_class.shape[0] / n_samples   # P(class)
            self.means[c]    = X_class.mean(axis=0)            # mean per feature
            self.variances[c]= X_class.var(axis=0)             # var per feature

    def _gaussian_pdf(self, x_value, mean, variance):
        variance    = variance + 1e-9                           # avoid div/0
        coefficient = 1 / np.sqrt(2 * np.pi * variance)
        exponent    = np.exp(-((x_value - mean) ** 2) / (2 * variance))
        return coefficient * exponent

    def _predict_one(self, x):
        posteriors = {}
        for c in self.classes:
            prior      = self.priors[c]
            likelihood = 1.0
            for feat_idx in range(len(x)):
                prob        = self._gaussian_pdf(x[feat_idx],
                                                  self.means[c][feat_idx],
                                                  self.variances[c][feat_idx])
                likelihood  = likelihood * prob   # independence assumption
            posteriors[c] = likelihood * prior

        return max(posteriors, key=lambda c: posteriors[c])   # highest posterior

    def predict(self, X):
        return np.array([self._predict_one(X[i]) for i in range(len(X))])

    def score(self, X, y):
        return (self.predict(X) == y).sum() / len(y)
Multiplying many small probabilities causes numerical underflow (→ 0). In production, use log probabilities: log_posterior = log(prior) + Σlog(feature_probs). Always mention this in interviews.

The Naive Assumption

P(x₁, x₂ | class) = P(x₁|class) × P(x₂|class) — features assumed independent given class. Almost never literally true but works surprisingly well in practice, especially for text.

Formula Reference Sheet

Total Error Decomposition

$$\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

Used when: Diagnosing underfitting vs overfitting

MSE — Mean Squared Error

$$\text{MSE} = \frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)^2$$

Used when: Regression loss; large errors are costly

Log Loss — Binary Cross-Entropy

$$L = -\frac{1}{n}\sum[y_i\log\hat{y}_i + (1-y_i)\log(1-\hat{y}_i)]$$

Used when: Binary classification training loss

Gradient Descent Update

$$\theta = \theta - \alpha \cdot \nabla L(\theta)$$

Variables: θ=weights, α=learning rate, ∇L=gradient of loss

Sigmoid Function

$$\sigma(z) = \frac{1}{1+e^{-z}} \qquad \frac{d\sigma}{dz} = \sigma(z)(1-\sigma(z)) = \hat{y}(1-\hat{y})$$

Used when: Logistic regression forward pass; squashes any number to (0,1)

Ridge Penalty (L2)

$$\text{Loss} = \text{MSE} + \lambda\sum\theta_j^2$$

Variables: λ=regularisation strength. In sklearn: alpha=λ, C=1/λ

Lasso Penalty (L1)

$$\text{Loss} = \text{MSE} + \lambda\sum|\theta_j|$$

Variables: Fixed-step gradient sign(θ) pushes weights to exactly zero

Normal Equation

$$\theta = (X^TX)^{-1}X^Ty$$

Used when: Small feature count — direct closed-form solution, no iterations

R² — Coefficient of Determination

$$R^2 = 1 - \frac{SS_{res}}{SS_{tot}} = 1 - \frac{\sum(y-\hat{y})^2}{\sum(y-\bar{y})^2}$$

Used when: Evaluating regression models. 1=perfect, 0=predicts mean, <0=broken

Entropy

$$H = -\sum_i p_i \log_2(p_i)$$

Used when: Decision tree split criterion. 0=pure, 1=maximum disorder

Information Gain

$$IG = H(\text{parent}) - \frac{n_L}{n}H(\text{left}) - \frac{n_R}{n}H(\text{right})$$

Used when: Choosing best split in Decision Tree — maximise IG

Euclidean Distance (KNN)

$$d(a,b) = \sqrt{\sum_i(a_i - b_i)^2}$$

Used when: KNN — finding nearest neighbours

Bayes Theorem (Naive Bayes)

$$P(\text{class}|\mathbf{x}) \propto P(\mathbf{x}|\text{class}) \times P(\text{class}) = \prod_j P(x_j|\text{class}) \times P(\text{class})$$

Used when: Naive Bayes prediction — pick class with highest posterior

Gaussian PDF (Naive Bayes)

$$P(x_j|\text{class}) = \frac{1}{\sqrt{2\pi\sigma^2}}\exp\!\left(-\frac{(x_j-\mu)^2}{2\sigma^2}\right)$$

Variables: μ=mean of feature j in class, σ²=variance of feature j in class

Precision, Recall, F1

$$\text{Precision} = \frac{TP}{TP+FP} \quad \text{Recall} = \frac{TP}{TP+FN} \quad F1 = \frac{2 \times P \times R}{P + R}$$

Used when: Classification with imbalanced classes or asymmetric error costs
